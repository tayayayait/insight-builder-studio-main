# 1. 문서 목적 및 범위
본 문서는 **AI OCR 기반 설문지 인식 및 자동 통계 분석 서비스**의 UI/UX를 개발·디자인이 그대로 참고하여 구현할 수 있도록 **규칙(Design System) + 화면/플로우 명세**를 정의한다.

- 대상: 기획/디자인/프런트엔드/백엔드(화면 연동)/QA
- 포함: 색상·타이포·레이아웃·컴포넌트 규격, 반응형 규칙, 주요 화면 구조, 상호작용(상태/에러/로딩), 접근성
- 제외: 모델 성능/통계 알고리즘 구현 상세(단, UI에서 노출되는 입력/출력 요구는 포함)

---

# 2. UX 원칙(필수 준수)
1) **실수 방지(오류 예방)**: 업로드/분석/내보내기 등 되돌리기 어려운 작업은 확인 모달 + 작업 단위(프로젝트/배치) 명확화.  
2) **신뢰(감사 가능성)**: OCR 추출값은 “원본 근거(좌표/페이지)”를 항상 확인 가능해야 하며, 수정 이력과 확정 상태를 표시.  
3) **효율(대량 처리)**: 목록·테이블·필터·일괄작업(배치/다중선택) 제공.  
4) **가시성(진행 상태)**: OCR/분석은 비동기 작업이므로 Job 진행률/대기열/완료 알림을 제공.  
5) **일관성**: 동일 개념은 동일 컴포넌트/라벨/아이콘/동작으로 통일(예: “프로젝트=설문 단위”, “배치=업로드 묶음”).

## 4.3 레이아웃/그리드/간격
### 4.3.1 브레이크포인트(반응형)
- `xs`: 0–359
- `sm`: 360–767 (대부분 모바일)
- `md`: 768–1023 (태블릿)
- `lg`: 1024–1279 (노트북)
- `xl`: 1280+ (데스크톱)

### 4.3.2 그리드
- Desktop(`lg+`): 12 columns, gutter 24px, 좌/우 margin 24px
- Tablet(`md`): 8 columns, gutter 20px, margin 20px
- Mobile(`sm-`): 4 columns, gutter 16px, margin 16px
- 최대 콘텐츠 폭: 1200px(리스트/대시보드), 검수 화면은 1440px까지 허용(뷰어+패널)

### 4.3.3 간격(Spacing Scale)
기본 단위 4px. 아래 값만 사용:
- `4, 8, 12, 16, 20, 24, 32, 40, 48, 64`

## 4.4 라운드/그림자/테두리
### 4.4.1 라운드(Radius)
- `--r-sm`: 8px (입력창, 작은 카드)
- `--r-md`: 12px (기본 카드/패널)
- `--r-lg`: 16px (모달)
- `--r-pill`: 999px (칩/배지)

### 4.4.2 보더(Border)
- 기본: 1px solid `--c-border`
- 강조: 1px solid `--c-border-strong`
- 포커스: 2px solid `--c-focus` (outline로 처리)

### 4.4.3 그림자(Elevation)
- `--e-1`: `0 1px 2px rgba(15,23,42,.08)` (카드 기본)
- `--e-2`: `0 6px 20px rgba(15,23,42,.12)` (드롭다운/팝오버)

# 5. 컴포넌트 규격(필수 공통 규칙)
> 모든 컴포넌트는 **상태(기본/hover/active/disabled/focus/error/loading)**를 지원해야 한다.

## 5.1 버튼(Button)
### 5.1.1 타입
- Primary: 주요 CTA(저장, 분석 실행, 업로드 완료)
- Secondary: 보조(미리보기, 필터)
- Tertiary/Ghost: 카드 내부 보조 액션(“자세히”)
- Danger: 삭제/되돌릴 수 없는 작업

### 5.1.2 크기
| 사이즈 | 높이 | 좌우 패딩 | 폰트 | 라운드 |
|---|---:|---:|---|---:|
| S | 32 | 12 | 12/18 | 8 |
| M(기본) | 40 | 16 | 14/22 | 10 |
| L | 48 | 20 | 14/22 | 12 |

### 5.1.3 상태 규칙
- Disabled: 배경 `--c-border`, 텍스트 `--c-text-muted`, 커서 not-allowed
- Loading: 스피너 + 텍스트 유지(폭 고정), 중복 클릭 방지

## 5.2 입력창(Input / Textarea)
### 5.2.1 기본 규격
- 높이: 40px(기본), 32px(S), 48px(L)
- 패딩: 12px
- 라운드: 10px
- 보더: 1px `--c-border`, focus 시 `--c-focus` 2px outline
- Placeholder: `--c-text-muted`

### 5.2.2 라벨/도움말/에러
- 라벨: 입력 상단 좌측, 12px~14px, 필수는 `*`(Danger 색)
- 도움말: 라벨 하단 또는 입력 하단, Small(12px), `--c-text-muted`
- 에러: 입력 하단, Small(12px), `--c-danger-600` + 아이콘, 입력 보더 `--c-danger-600`

## 5.3 셀렉트(Select) / 콤보박스
- 기본 높이 40px, 우측 드롭다운 아이콘
- 검색형 콤보박스는 옵션이 20개 이상이거나 “프로젝트/변수” 선택에 사용
- 옵션 리스트 최대 높이: 320px, 스크롤

## 5.4 체크박스/라디오/토글
### 체크박스
- 크기 16px, 라운드 4px
- 표/리스트에서 다중선택(배치/행 선택)
### 라디오
- 크기 16px
- 단일 선택(분석 그룹 지정 등)
### 토글
- 크기: 36×20px, 손잡이 16px
- 설정(자동 확정, 알림 수신)

## 5.5 배지/칩(Badge/Chip)
### 5.5.1 배지(상태 표시)
| 상태 | 배경 | 텍스트 |
|---|---|---|
| 완료 | `--c-success-50` | `--c-success-600` |
| 진행중 | `--c-primary-50` | `--c-primary-600` |
| 경고 | `--c-warning-50` | `--c-warning-600` |
| 오류 | `--c-danger-50` | `--c-danger-600` |

### 5.5.2 칩(필터 토큰)
- 높이 28px, 패딩 10px, 라운드 pill
- 좌측 라벨 + 우측 “x” 제거 아이콘

## 5.6 카드(Card)
### 5.6.1 기본 카드
- 배경 `--c-surface`, 보더 1px `--c-border`, 라운드 12px, 그림자 `--e-1`
- 내부 패딩: 16px(기본), 섹션 구분은 16px 간격
### 5.6.2 통계 KPI 카드
- 높이 최소 96px
- 좌: KPI 라벨(12px), 값(24px/700), 우: 증감/아이콘
- 클릭 시 상세로 Drill-down 가능(옵션)

## 5.7 테이블(Table)
### 5.7.1 규격
- 헤더 높이 44px, 바디 행 높이 44px
- 셀 패딩: 좌우 12px, 상하 10px
- 헤더 배경: `--c-bg`, 보더 하단 `--c-border-strong`
- 정렬: 숫자 우측 정렬, 텍스트 좌측 정렬, 상태/배지는 중앙 가능

### 5.7.2 기능 요구
- 정렬(sort), 페이지네이션, 컬럼 숨김/표시, 열 너비 조절(검수 화면 우선)
- 행 선택(checkbox) + 일괄 작업(삭제, 재분석, 내보내기)
- 빈 상태/로딩 스켈레톤 제공

## 5.8 탭(Tabs)
- 높이 44px
- 활성 탭: 텍스트 `--c-primary-600` + 하단 인디케이터 2px
- 예: “요약/상세/원본/로그”

## 5.9 모달(Modal)
### 5.9.1 크기
| 타입 | 폭 | 사용처 |
|---|---:|---|
| S | 420px | 확인/경고 |
| M | 640px | 폼(프로젝트 생성) |
| L | 960px | 내보내기 설정/미리보기 |

### 5.9.2 규칙
- 오버레이: `--c-overlay`
- 닫기: 우상단 X + ESC + 오버레이 클릭(파괴적 작업 시 오버레이 클릭 비활성)
- 하단 버튼 영역: Primary(우측) / Secondary(좌측)

## 5.10 드로어/사이드패널(Drawer)
- 우측에서 슬라이드(검수/상세)
- 폭: 360px(기본), 480px(상세 편집)
- 스크롤: 본문만 스크롤, 헤더/푸터 고정

## 5.11 토스트/알림(Toast)
- 우측 상단 스택(최대 3개 표시)
- 자동 닫힘 4초(오류는 8초 또는 수동)
- Job 완료 알림은 “바로가기” 액션 포함

## 5.12 파일 업로드(Uploader)
- 드래그&드롭 영역 높이 160px(Desktop), 120px(Mobile)
- 허용: PDF, JPG/PNG, XLSX/CSV
- 업로드 전 검증: 파일 확장자/용량(기본 50MB, 설정 가능), 페이지 수(예: 200p 제한)
- 업로드 진행률 + 실패 시 재시도

## 5.13 PDF/이미지 뷰어(검수 화면 핵심)
- 중앙 뷰어 + 좌측 썸네일 + 우측 필드 목록(3-pane)
- 줌: 50–200%, 10% step + “맞춤/너비맞춤”
- 페이지 이동: 상단 페이지네이션 + 키보드(← → / PgUp PgDn)
- OCR 하이라이트: 필드 박스(Primary 30% 투명), 선택 시 60% + 테두리 강조

# 6. 공통 패턴(화면 규칙)
## 6.1 페이지 헤더
- 좌: 페이지 타이틀(H1) + 보조 설명(Small)
- 우: 주요 액션(Primary), 보조 액션(Secondary), 도움말 아이콘
- 브레드크럼(선택): 프로젝트/배치 계층이 깊은 화면에만 표시

## 6.2 필터 바(목록 공통)
- 구성: 기간(날짜 범위) / 상태 / 검색어 / 고급필터(드로어)
- 적용 방식: “자동 적용” 기본 + 고급필터는 “적용” 버튼 제공(복잡도에 따라)
- 필터는 칩으로 노출(제거 가능)

## 6.3 빈 상태(Empty State)
- 아이콘 + 1줄 설명 + Primary 버튼(첫 행동 유도)
- 예: “업로드된 설문지가 없습니다. PDF를 업로드해 OCR을 시작하세요.”

## 6.4 에러 처리
- 필드 에러: 입력 하단 인라인
- 작업 에러: 페이지 상단 Alert + 재시도 버튼
- 시스템 장애: “상태 페이지/문의” 링크(옵션)

## 6.5 로딩/비동기 작업(Job)
- 목록/테이블: 스켈레톤
- 버튼: 내부 스피너
- OCR/분석: Job 카드(진행률 바, ETA, 단계 표시: 업로드→OCR→정제→분석→리포트)
- 작업 중 페이지 이탈 가능, 완료 시 알림/메일(옵션)

# 7. 화면 명세(PC 우선, 모바일 규칙 포함)
> 화면은 “프로젝트(설문 단위) → 배치(업로드 묶음) → 결과(데이터/분석/리포트)” 흐름으로 설계한다.

## 7.1 로그인/권한
### 화면 구성
- 좌측: 제품 소개(일러스트/문구), 우측: 로그인 폼 카드(폭 420px)
- 입력: 이메일/비밀번호, 로그인 버튼(Primary), 비밀번호 보기 토글
### 규칙
- 실패 시: “이메일 또는 비밀번호가 올바르지 않습니다.” 인라인 + 토스트
- 계정 상태(잠김/비활성) 메시지 분리

## 7.2 대시보드
### 상단 KPI(카드 4개)
- 오늘 업로드 수 / 처리중 배치 / 오류 항목 / 최근 리포트
### 메인 영역
- “최근 작업” 테이블: 배치명, 프로젝트, 상태, 진행률, 생성일, 액션(보기)
- “알림” 패널: 실패/주의 이벤트(재시도/상세)
### 모바일
- KPI 2열 그리드, 테이블은 카드 리스트로 전환

## 7.3 프로젝트 목록/생성
### 목록
- 테이블 컬럼: 프로젝트명, 설명, 생성자, 최근 업데이트, 배치 수, 상태
- 상단: 검색 + “프로젝트 생성”(Primary)
### 생성/편집 모달(M)
- 필드: 프로젝트명(필수), 설명, 설문 유형(선택: PDF 기반/엑셀 기반/혼합), 기본 통계 옵션(토글)
- 생성 후: “업로드로 이동” CTA 제공

## 7.4 데이터 업로드/가져오기(수집)
### 레이아웃(2단)
- 좌: 업로더(드롭존) + 업로드 큐(파일 리스트)
- 우: 배치 설정 카드
  - 배치명(기본: 날짜+프로젝트명)
  - 데이터 유형: PDF/이미지/엑셀(복수 가능)
  - 설문 템플릿 매칭(자동/수동)
  - OCR 옵션: 언어, 체크박스 인식 강화, 손글씨(옵션)
  - 개인정보 포함 여부(토글) + 마스킹 옵션(선택)
### 제출
- “업로드 시작”(Primary) → Job 생성 → 작업 현황으로 이동

## 7.5 배치/작업 현황
### 테이블
- 배치명, 프로젝트, 파일 수, 페이지 수, 상태(배지), 진행률, 실패 건수, 생성일
- 행 클릭: 배치 상세(탭: 요약/파일/로그)
### 배치 상세
- 요약 KPI: 전체 응답지 수, 추출 항목 수, 미확정 항목, 오류율
- 파일 탭: 파일별 상태 + 재처리
- 로그 탭: 단계별 로그(접기), 오류 원인 코드 표시

## 7.6 OCR 결과 검수/수정(핵심 화면)
### 7.6.1 레이아웃(3-pane, Desktop)
- 좌측(썸네일): 페이지 리스트(폭 220px)
  - 각 썸네일: 페이지 번호 + 상태 점(정상/주의/오류)
  - 필터: “미확정만”
- 중앙(뷰어): PDF/이미지 캔버스
  - 상단 툴바: 줌, 페이지 이동, 하이라이트 토글, 회전, 밝기/대비(선택)
- 우측(필드 패널): 추출 필드 목록(폭 360~480px)
  - 필드 아이템: 문항명, 추출값, 신뢰도(%) 배지, 상태(확정/미확정/오류), 편집 아이콘
  - 필드 클릭 시: 중앙 하이라이트로 동기화

### 7.6.2 필드 편집(우측 패널 내)
- 타입별 UI
  - 숫자: 숫자 키패드(모바일), 단위 표시(옵션)
  - 텍스트/손글씨: 텍스트 입력 + “원본 보기”(확대 팝오버)
  - 체크박스: 단일/복수 체크 UI, “전체 해제” 제공
- 편집 시: 변경 전/후 표시(작게), “되돌리기” 제공
- 저장 방식: 자동 저장(기본) + 상단에 “저장됨” 토스트 / 실패 시 재시도

### 7.6.3 확정(Validate) 규칙
- 필드 상태
  - 확정: 사용자가 확인 완료
  - 미확정: 신뢰도 낮음 또는 미검수
  - 오류: 파싱 실패/형식 불일치(예: 숫자 문항에 문자)
- 배치 확정 조건(권장)
  - 미확정 필드가 0이거나, 관리자가 “미확정 포함 분석 허용” 토글을 켠 경우만 분석 실행 가능

### 7.6.4 검수 생산성 기능
- 키보드 단축키(Desktop)
  - `J/K`: 다음/이전 필드
  - `N/P`: 다음/이전 페이지
  - `Enter`: 확정 토글
  - `Ctrl+Z`: 마지막 수정 되돌리기
- “미확정 큐” 모드: 미확정 항목만 순차 이동

### 모바일
- 2-pane 전환: 상단 탭(뷰어/필드) 또는 드로어로 필드 패널 표시
- 썸네일은 하단 슬라이더로 축약

## 7.7 통계 분석(실행/결과)
### 7.7.1 분석 실행 화면
- 상단: 대상 선택(프로젝트/배치/엑셀 데이터셋)
- 분석 옵션 카드
  - 기본 집계: 응답자 수, 평균/표준편차
  - 상관관계: 변수 선택(다중), 피어슨/스피어만
  - t-test: 그룹 변수 선택 + paired 여부 토글
  - IPA: 중요도/만족도(또는 성과) 매핑 규칙 선택
- 실행 버튼(Primary): “분석 실행”
- 유효성
  - 최소 표본수 미달 시 경고(Alert) + 실행은 관리자만 허용(옵션)

### 7.7.2 결과 화면(탭 구조)
탭: `요약` / `상관관계` / `t-test` / `IPA` / `데이터`

#### 요약
- KPI: 응답자 수, 문항 수, 결측치 비율, 최종 확정률
- 표: 문항별 평균/표준편차/응답 분포(Top/Bottom)
#### 상관관계
- 상관 행렬(히트맵) + 상위 상관 Top10 리스트
- 셀 클릭 시: 산점도 + 회귀선(옵션) + 데이터 포인트 수
#### t-test
- 테이블: 그룹 A/B(또는 전/후), 평균, 차이, t, p-value, 효과크기(선택)
- p-value는 임계(0.05) 기준 색상 강조(주의: 과도한 강조 금지, “참고용” 문구)
#### IPA
- 2×2 사분면 차트(중요도 vs 성과)
- 점 클릭 시: 문항 상세(응답 분포/개선 제안 메모 입력)
#### 데이터
- 정형 데이터 테이블 + “엑셀 내보내기” 버튼

### 모바일
- 차트는 세로 스택, 표는 가로 스크롤 또는 카드로 전환

## 7.8 리포트(문서화/내보내기)
### 리포트 생성(마법사 Stepper)
1) 대상 선택(분석 결과 선택)
2) 템플릿 선택(기본/상세/관리자용)
3) 포함 항목 선택(체크박스): 요약/상관/t-test/IPA/원자료/부록
4) 커버 정보: 제목, 기간, 작성자, 로고 업로드(옵션)
5) 미리보기(페이지 썸네일) + 생성

### 내보내기 모달(L)
- 포맷: PDF / PPTX / XLSX
- 옵션: 표 글자 크기, 차트 색상(기본/고대비), 개인정보 마스킹
- 생성 상태: 진행률 + 완료 시 다운로드 버튼

## 7.9 관리자 페이지(권한/검수/정책)
### 사용자/권한
- 테이블: 이름, 이메일, 역할(Admin/Manager/Reviewer/Viewer), 상태
- 역할 권한 매트릭스(읽기/쓰기/삭제/내보내기/관리자)
### 설문 데이터 검수 정책
- “미확정 포함 분석 허용” 토글(프로젝트 단위)
- “신뢰도 임계값” 설정(예: 0.85 미만은 미확정)
- 감사 로그: 사용자/행동/대상/시간/IP

# 8. 반응형/디바이스 규칙(필수)
## 8.1 사이드바 동작
- `lg+`: 좌측 고정(폭 264px), 접기 가능(아이콘 모드 80px)
- `md-`: 드로어(오버레이), 메뉴 선택 후 자동 닫힘

## 8.2 테이블 변환
- `sm-`: 테이블 → 카드 리스트(핵심 필드 3개 + “더보기”)
- 데이터 테이블은 가로 스크롤 허용(대신 고정 컬럼 최소화)

## 8.3 검수 화면
- `lg+`: 3-pane 유지
- `md`: 2-pane(썸네일 축소 + 우측 패널 탭)
- `sm-`: 뷰어 단독 + 필드 패널 드로어

# 9. 접근성(A11y) / 사용성(필수)
- 대비: 텍스트 대비 최소 4.5:1(Body), 큰 텍스트 3:1
- 키보드: 모든 인터랙션 요소 탭 이동 가능, 포커스 링 항상 표시(`--c-focus`)
- 폼: label 연결(`for/id`), 에러는 aria-describedby로 연결
- 차트: 요약 테이블(데이터) 제공 + 색상만으로 의미 전달 금지(패턴/레이블 병행)
- 시간/숫자: 로케일 `ko-KR`, 천 단위 구분(예: 12,345)

# 10. Figma/개발 핸드오프 규칙(권장)
- 컴포넌트 네이밍: `Component/Variant/Size/State` (예: `Button/Primary/M/Loading`)
- 토큰 우선: 색/간격/라운드/그림자는 토큰으로만 지정(하드코딩 금지)
- 아이콘: 20px(기본), 16px(소), 24px(대) 3종만 사용
- 스페이싱: 4px 스케일만 사용

# 11. 문구(마이크로카피) 규칙(일관성)
- Primary CTA는 동사로 시작: “업로드 시작”, “분석 실행”, “리포트 생성”
- 위험 작업은 명확히: “삭제”, “영구 삭제”, “되돌릴 수 없음” 문구 포함
- 검수 상태 용어 고정: “확정/미확정/오류”
